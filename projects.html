Running Log

I decided to build a running log during the pandemic as a way to stay sane and reduce my likelihood of overtraining. It is written in Python3 where users can add runs with distances, times, time of day, date, and comments. This data is stored in a .csv file which is sorted in order by date whenever users add new runs. Because we have an already sorted file, insertion sort is use after a user adds a run.

Features

  • Adding runs and being able to switch between imperial and metric units
  • Fully interactive calendar that displays distance, time, and pace for each day in a month
    • Users can then click on the date in the calendar to see more info
  • Training load status. Determine if training is enough and if there's a risk for injury
    • Sleep data is scraped from Garmin API

House automation

A variety of projects for my college house that improves the quality of life. Almost all these projects are run using Raspberry Pi's

  • Sound request station
    • Touchscreen jukebox where people are able to control the music on the main floor
    • Filter added to block songs. Funny messages are then displayed based on who I think requested it
  • Card swipe access to house locks
    • Using our URID cards, we can swipe, using an electromagnetic card reader, into the house, unlocking or locking the door
    • A servo is used to control the rods that rotate the deadlock.
    • Mechanical locking is supported as the program can recognize if the state of the lock is different from last usage.

Machine independent optimized C compiler

For a subset of the C programming language, I built a compiler that scans, parses, builds intermediate representations, and optimizes the code. Scanner, parser, and abstract syntax tree (AST) are built using C++. 3 address code and all optimizations are implemented using Python3.

Features

  • Scanner that finds syntax errors
  • Parser that builds an abstract syntax tree (AST) Error recovery is implented using Wirth's algorithm so that after an error, the program will recover to find additional errors that are displayed to user.
  • From AST, 3 address code is generated, which is machine independent, not control flow bound low level code
  • Using 3 address code, our basic blocks and control flow graph are generated to perform optimizations and thus data flow analysis

Optimization methods used

  • Local Value Numbering
  • Dead Code elimination
  • Partial redunancy elimination

File compression

Class project in Java where given a file of any extension to reduce its memory storage. Huffman encoding algorithm is used to reduce the storage of upwards of 100% and usually 50%. Project also enables the user to decompress the file and see its original version.
