Running Log

I decided to build a running log during the pandemic as a way to stay sane and reduce my likeihood of overtraining. It is written in Python3 where users can add runs with distances, times, time of day, date, and comments. This data is stored in a csv which is kept stored sorted in order by date whenever users add new runs. Because we have an already sorted insertion sort is use after a user adds a run.

Features

  • Adding runs and being able to switch between imperial and metric units
  • Fully interactive calendar that displays distance, time, and pace for each day in a month
    • Users can then click on the date in the calendar to see more info
  • Training load status. Determine if training is enough and if there's a risk for injury
    • Sleep data is scraped from Garmin API

File compression

Class project in Java where given a file of any extension to reduce its memory storage. Huffman encoding algorithm is used to reduce the storage of upwards of 100% and usually 50%. Project also enables the user to decompress the file and see its original version.

Regular expression parser

Class project that parses a regular expression and determines if it is valid. Table-driven and recursive descent parsing algorithms written in C are both used.
